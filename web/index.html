Help & Documentation

What is TurboQuant?

TurboQuant (Zandieh et al., ICLR 2026) is Google's breakthrough method for compressing LLM KV caches from FP16 to 3 bits per value, achieving 4.9x compression with near-zero accuracy loss (MSE 0.034).

How it works

  1. Random Orthogonal Rotation — Each KV vector is normalized and rotated by a fixed random orthogonal matrix, making all coordinates follow the same Beta distribution.
  2. Lloyd-Max Quantization — Each rotated coordinate is quantized to 3 bits using an MSE-optimal codebook for that distribution.
  3. Bit Packing — 128 indices packed into 48 bytes (TQ3) or 64 bytes (TQ4), plus 4 bytes for the norm.
  4. Dequantization — Reverse: unpack → codebook lookup → inverse rotation → scale by norm.

Why it matters

On a 4GB GPU (RTX 3050), with FP16 KV cache you might fit ~8K context. With TQ3, the same VRAM holds ~40K tokens — 5x more context for the same memory cost.

Optimization Checklist

GPU Optimization

  • Set gpu_layers = 99 to offload all layers to GPU
  • If VRAM is tight, reduce layers until it fits (check GPU memory in status bar)
  • Use Q4_K_M for best quality/size balance on 4GB VRAM
  • Use IQ2_XXS or TQ2_0 to fit larger models in limited VRAM

Context & Memory

  • Default context: 4096 tokens. Increase to 8192+ for long conversations
  • KV cache uses ~2MB per 1K tokens at FP16; TQ3 reduces this to ~0.4MB
  • For very long contexts (32K+), TQ3 KV cache compression is essential

System Tuning

  • Linux: echo madvise | sudo tee /sys/kernel/mm/transparent_hugepage/enabled
  • Set CPU governor to performance: cpupower frequency-set -g performance
  • Disable swap or use zswap for better memory performance

Connect from Windsurf / VSCode

Set the Ollama API endpoint to http://localhost:11434 in your IDE settings.

curl Examples

# Chat (streaming)
curl http://localhost:11434/api/chat -d '{
  "model": "SmolLM2-1.7B-Instruct-Q4_K_M",
  "messages": [{"role":"user","content":"Hello!"}]
}'

# Non-streaming
curl -X POST http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"SmolLM2-1.7B-Instruct-Q4_K_M",
       "messages":[{"role":"user","content":"Hello!"}]}'
